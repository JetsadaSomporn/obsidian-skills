# AI Handover Guide (Read This First)

> ยินดีต้อนรับ AI Agent เพื่อนร่วมงาน! นี่คือคู่มือการใช้สมองที่สองของเจ้าของ Vault นี้

## 🧠 Knowledge Structure
Vault นี้จัดระเบียบตามหลัก **PARA + Atomic Notes**:
- **Tech/Atomic:** เก็บ Pattern โค้ดที่ผ่านการทดสอบแล้ว (Verified)
- **Concepts:** เก็บคำนิยามเชิงทฤษฎี (The "Why")
- **Projects:** งานที่กำลังทำอยู่และสถาปัตยกรรมของโปรเจกต์ (เช่น STS, Stock Manager)
- **Identity:** ตัวตน, เป้าหมาย, และหลักการตัดสินใจ (Operating Principles)

## 🎯 How to Help the Owner
1. **Reuse First:** ก่อนเสนอโค้ดใหม่ ให้เช็กใน `Tech/Atomic` หรือ `Tech/MOC - Tech Overview` ก่อนเสมอว่ามี Pattern ที่เจ้าของเคยใช้ไหม
2. **Be Atomic:** ถ้าจะบันทึกความรู้ใหม่ ให้แตกเป็น Node เล็กๆ (Atomic) และเชื่อมโยง (Link) กับ Node ที่มีอยู่เสมอ
3. **Check Constraints:** ก่อนแก้ไขโปรเจกต์ ให้ดู `Identity/Operating Principles` เพื่อเข้าใจข้อจำกัดหรือสไตล์ที่เจ้าของต้องการ (เช่น การปิด HTTPS Redirection ในบางโปรเจกต์)
4. **Link Everything:** อย่าสร้างเกาะกำบัง (Island) ทุก Node ต้องเชื่อมไปหา MOC หรือ Node อื่นที่เกี่ยวข้อง
