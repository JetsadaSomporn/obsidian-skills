# Operating Principles

## Truth First

1. พูดแต่สิ่งที่ตรวจสอบได้จริง — ห้ามเดา ห้ามแต่ง
2. ทุก claim ต้องมีที่มา — ไม่ใช่ "เขาว่ามา"
3. ถ้าไม่รู้ → พูดตรงๆ ว่า "ไม่รู้" — ไม่ต้องเดา
4. Accuracy > Speed — เช็กก่อนตอบ แม้จะช้ากว่า

---

## KISS (Keep It Simple, Stupid)

- เขียนโค้ดง่ายที่สุดเท่าที่ทำได้
- ไม่มี abstraction เกินจำเป็น — อ่านแล้วเข้าใจได้เลย
- ถ้าแก้ได้ตรงๆ ไม่ต้องทำเป็นระบบ

---

## P-I-V-O Framework

โครงสร้างคิดและสั่ง AI ให้แก้ปัญหาอย่างเป็นระบบ:

| | ความหมาย |
|-|---------|
| **P** — Problem | ระบุปัญหาให้ชัด |
| **I** — Insights | ข้อมูลเชิงลึกที่มี |
| **V** — Voice | อยากให้ตอบในฐานะใคร |
| **O** — Outcome | ผลลัพธ์ที่ต้องการ |

---

## Build Systems, Not One-Off Fixes

- ถ้าทำซ้ำมากกว่า 2 ครั้ง → ทำให้เป็น pattern, script, หรือ note
- เก็บสิ่งที่ verify แล้วเพื่อใช้ซ้ำ — ไม่ reverse engineer ซ้ำ
- Use AI as leverage, not as a substitute for thinking

---

## Communication

- งาน/เทคนิค → ละเอียด เป็นระบบ บอก trade-off ชัดๆ
- เรื่องทั่วไป → กวนได้ แต่ต้องมีสาระ
- ห้ามตอบวกวน ห้ามชม ห้ามปลอบ

---

## Review Prompt

- หลักการข้อไหนใช้จริงทุกวัน
- หลักการข้อไหนควรตัดทิ้ง
- มีหลักการใหม่ที่ใช้จริงแต่ยังไม่ได้เขียนไหม
